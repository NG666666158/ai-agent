# Orion Agent 产品路线图

## 1. 路线图定位

这份路线图回答的是：

- 这个项目从产品形态上要走到哪里
- 当前阶段已经完成了什么
- 后续应该如何在保留 Web 可视化 Agent 体验的前提下继续升级

本路线图不单独讨论底层内核细节；内核路线请同时参考：

- [enterprise-stability-roadmap-v1.md](C:/github/ai-agent/ai-agent/docs/enterprise-stability-roadmap-v1.md)

---

## 2. 产品目标

Orion Agent 的产品目标不是普通聊天页面，而是一个：

> 让用户可以在 Web 页面中直接看到“任务是如何被理解、如何被检索、如何被执行、为什么给出这个结果”的可视化 Agent 工作台

这个目标意味着必须长期保留并持续增强以下产品特征：

- 左侧历史会话，右侧当前会话的标准 Agent 工作台布局
- 最终回答与系统思考过程分区展示
- 思考过程可折叠
- 流式回答与流式系统进度
- Markdown 结果展示
- 来源锚点、脚注与命中说明
- 会话、记忆、画像、任务追溯的可见性

---

## 3. 当前已完成的产品里程碑

截至 `2026-04-18`，当前版本已经完成以下主体里程碑：

- 单任务执行闭环
- 流式系统进度与流式回答正文
- 可折叠思考过程
- 任务详情、工具调用详情与结果导出
- Chat session、历史记录、上下文压缩
- 长期记忆管理页
- 用户画像页、画像编辑与冲突合并
- 回答来源提示、段落锚点、引用脚注
- 自动化测试与中文测试报告
- 控制台主界面逐步收敛为 Agent 对话布局

这说明项目已经完成了：

> 从“功能演示型原型”到“可视化 Agent 工作台原型”的跨越

---

## 4. 当前产品阶段判断

当前产品阶段更适合定义为：

- 已经具备可演示、可测试、可持续重构的 Agent 产品雏形
- 仍未达到真正的平台级、组织级可用产品状态

当前最重要的问题已不再是“页面够不够多”，而是：

- 思考过程和内核真相是否一致
- 跨会话记忆和画像是否稳定
- 来源解释是否可信
- 恢复流是否可理解
- 多用户与权限体系是否具备扩展空间

---

## 5. 下一阶段产品路线

### 路线 A：执行过程体验升级

目标：

- 让用户不只看到“步骤列表”，而是真正看懂 Agent 执行链路

重点：

- 更细颗粒的阶段说明
- 更流畅的系统进度更新
- 更稳定的执行时间线
- 工具调用、检索、恢复、重规划的统一表现

阶段价值：

- 提升系统可信度
- 让可视化体验真正体现 Agent 特征

### 路线 B：来源解释与记忆体验升级

目标：

- 让“为什么这样回答”具备更强可解释性

重点：

- 后端稳定输出 citation map
- 段落与记忆、画像、历史消息做稳定映射
- 记忆命中原因更明确
- 画像与偏好展示更可读

阶段价值：

- 提高回答可信度
- 提高跨会话连续体验

### 路线 C：会话产品化升级

目标：

- 让会话成为长期工作的主入口，而不是临时聊天记录

重点：

- 会话追溯能力
- 分支续聊
- 上下文摘要刷新
- 当前会话与历史会话间的关系可视化

阶段价值：

- 提升用户“长期使用”的连续感

### 路线 D：用户与组织级能力

目标：

- 让系统逐步具备真正的平台级使用能力

重点：

- 多用户隔离
- 权限确认
- 审计日志
- 部署与运行治理

阶段价值：

- 为企业级产品形态打底

---

## 6. 产品路线优先级

建议按以下顺序推进：

1. 执行过程体验升级
2. 来源解释与记忆体验升级
3. 会话产品化升级
4. 用户与组织级能力

这个顺序的原因是：

- 当前项目最大的产品价值来自“可视化 Agent 执行体验”
- 只有先把执行链路、来源解释和跨会话连续性做稳，后续多用户和企业化能力才有承接基础

---

## 7. 产品里程碑定义

### M1：可理解的 Agent 执行体验

验收标准：

- 用户可以从控制台清楚看到任务解析、检索、工具调用、恢复和回答生成
- 思考过程可折叠但信息完整
- 系统进度与回答生成体验一致

### M2：可解释的回答与记忆利用

验收标准：

- 用户可以知道回答用了哪些来源
- 用户可以追到具体是哪条记忆、画像或历史消息影响了回答
- 跨会话偏好能够稳定生效

### M3：可持续使用的会话产品

验收标准：

- 会话可以长期保留
- 历史追溯、分支续聊、上下文摘要机制稳定
- 用户能理解新会话和旧会话之间的关系

### M4：可进入平台化阶段的产品底座

验收标准：

- 多用户边界已建立
- 权限确认有统一模型
- 部署与运行观测有基础方案

---

## 8. 与内核路线的关系

需要特别明确：

- 产品路线解决的是“用户看见什么、如何使用、是否顺畅可信”
- 内核路线解决的是“系统如何稳定、如何恢复、如何长期扩展”

两者必须同步推进，不能分离。

因此，后续所有产品层开发都应该受以下原则约束：

1. 前端展示不能脱离后端真实执行语义。
2. 思考链路必须尽量基于统一 runtime schema。
3. 引用、来源、恢复痕迹要以后端稳定产物为主。
4. 不再为了页面效果新增大量一次性临时字段。

---

## 9. 统一结论

Orion Agent 的产品路线，不是继续堆页面，而是：

> 把当前已经成形的 Web 可视化 Agent 工作台，继续做成“用户看得懂、来源讲得清、会话能延续、内核够稳定”的产品形态；再在此基础上逐步进入多用户、权限和生产化阶段。
