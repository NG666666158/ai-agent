# Orion Agent 产品路线图

## 当前状态

当前版本已经完成以下里程碑：

- 单任务执行闭环
- 流式系统进度与流式回答正文
- 高风险文件读取确认
- 任务详情、工具调用详情与结果导出
- Chat session、历史记录、上下文压缩
- 长期记忆管理页
- 用户画像页、画像编辑与冲突合并
- 回答来源提示、段落锚点、引用脚注
- 自动化测试与中文测试报告

这意味着项目已经完成了“可演示、可测试、可持续迭代原型”的阶段。

## 下一阶段路线

### 路线 A：执行内核增强

目标：

- checkpoint 更细
- 恢复能力更强
- 重规划更可解释
- 上下文分层更稳定

建议拆分：

1. checkpoint 细化为解析、记忆召回、规划、执行、反思等阶段
2. 区分工具失败、评审失败、恢复重跑、上下文刷新等重规划原因
3. 为上下文层建立长度预算和裁剪策略
4. 为后续结构化 citation map 预留数据位

### 路线 B：来源治理与记忆治理

目标：

- 回答为什么这样答更可解释
- 历史来源更可追溯
- 记忆和画像更可控

建议拆分：

1. 后端输出结构化 citation map
2. 回答段落和来源做稳定映射
3. 记忆审核、版本比较和回滚
4. 画像人工确认与冲突治理

### 路线 C：多用户隔离与权限体系

目标：

- 数据按用户隔离
- 权限模型成体系
- 高风险操作可审计

建议拆分：

1. 引入 `user_id`
2. 任务、会话、记忆、画像按用户归属
3. 角色权限和操作审计
4. 高风险工具权限矩阵

### 路线 D：生产化与运维能力

目标：

- 部署更稳定
- 指标更完整
- 故障恢复更可靠

建议拆分：

1. 环境配置分层
2. 数据库与向量库备份策略
3. 更完整的业务指标与告警
4. 容器资源约束与恢复策略

## 优先级建议

建议优先级如下：

1. 执行内核增强
2. 来源治理与记忆治理
3. 多用户隔离与权限体系
4. 生产化与运维能力

## 里程碑定义

### M1：执行内核可恢复

- 支持更细粒度 checkpoint
- 支持阶段恢复
- 重规划原因和失败分类清晰
- 上下文构建稳定，不因多轮对话明显退化

### M2：来源与记忆可解释

- 回答正文具备稳定来源映射
- 用户能从回答反查到记忆、画像和历史任务
- 记忆和画像的修改链条可追溯

### M3：系统可多用户使用

- 数据具备用户隔离
- 高风险动作具备权限控制
- 关键操作具备审计记录

### M4：系统可稳定部署

- 部署与监控流程标准化
- 数据可备份、可恢复
- 核心错误和运行指标可持续观测
